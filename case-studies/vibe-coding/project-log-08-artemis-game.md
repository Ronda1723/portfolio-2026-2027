## The story

**What problem does this solve?**
This one isn't a productivity tool — it's a celebration and a learning exercise. Artemis II launched on the same day Samita built this, and she wanted to mark the moment. She'd also never built a game before and wanted to explore a completely different technical paradigm.

**How did the idea come up?**
Samita wanted to try game development as a new concept. The Artemis II launch happening that day provided the perfect theme — timely, exciting, and makes the project feel intentional rather than random. "Artemis II launched today. I built a game to celebrate."

**What does it actually do?**
- Tap/click to make the space capsule fly upward
- Dodge obstacles (space debris / asteroid field)
- Score counter tracks how far you get
- Space-themed visuals matching the Artemis II mission aesthetic
- Simple, addictive Flappy Bird mechanic

---

## Decisions you made

**What's the most interesting design/product decision you made and why?**
Tying the game to a real-world event happening that day. This transforms a generic Flappy Bird clone into something timely and shareable. It also shows cultural awareness — the ability to connect what you're building to what's happening in the world.

**Did you consider an alternative approach and reject it? What and why?**
Considered more complex game ideas but deliberately chose Flappy Bird as the template — it's the simplest possible game mechanic (tap to fly, avoid obstacles). The learning goal was game development fundamentals (game loops, physics, collision detection), not game complexity. Start simple, understand the paradigm first.

**What would you do differently if you had more time?**
Power-ups, multiple levels, a leaderboard, particle effects on collision, sound design. Also: actually learning about game physics more deeply rather than relying on the AI tool to handle it.

---

## What was hard

**What broke, confused you, or took way longer than expected?**
Game development is a completely different paradigm from everything else in the challenge. Web apps are event-driven (user clicks → thing happens). Games are loop-driven (every frame, update everything). Understanding game loops, frame-by-frame rendering, collision detection boxes, and gravity physics was genuinely new territory.

**What technical concept did you learn today?**
Game development fundamentals: game loops, frame rendering, collision detection, tap/click physics, score tracking.

**Rate your understanding of that concept now (1-5):** 2 (built something, but the tool did most of the heavy lifting)
